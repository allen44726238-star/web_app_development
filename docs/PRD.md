# 線上算命系統 PRD

## 1. 專案概述

### 背景與動機
現代人常面臨生活、工作或感情上的抉擇，需要穩定心靈或尋求方向的指引。傳統上人們會前往廟宇求籤或找算命師，但受限於時間與空間的阻礙。本專案希望能建立一個「線上算命系統」，結合傳統命理文化與現代網頁技術，讓使用者不受時空限制，隨時隨地皆能進行求神問卜，尋求心靈上的平靜。

### 目標用戶
- 對傳統命理、占星、算卜感興趣的一般大眾。
- 面臨生活抉擇、需要尋求心靈寄託或指引的現代人。
- 工作繁忙而無法親自前往實體廟宇祭拜求籤的信徒。

### 核心價值主張
提供具備儀式感且操作簡單的數位算命體驗。不僅能讓大眾快速抽籤解惑，更結合數位化優勢，讓使用者可記錄與追蹤個人的抽籤歷史，並提供添香油錢的回饋機制，打造完整的線上命理服務。

---

## 2. 功能需求

### 主要功能清單與使用者故事

1. **算命、占卜、抽籤**
   - **作為** 使用者，**我希望** 能在網站上進行抽籤或占卜的操作，**以便** 當遇到難題時能即時獲得命理指引。
2. **靈籤解析與結果呈現**
   - **作為** 使用者，**我希望** 抽籤後能獲得清晰的運勢說明與詳細解籤（如事業、感情分析），**以便** 將傳統籤詩化為白話文，正確了解當前的吉凶與建議。
3. **儲存算命結果**
   - **作為** 註冊會員，**我希望** 系統能夠自動保存我過去每一次的算命與抽籤紀錄，**以便** 日後可以回顧這段時間的運勢變化或印證結果。
4. **捐香油錢 (線上贊助)**
   - **作為** 滿意系統服務的虔誠使用者，**我希望** 有管道可以進行線上捐款給系統或合作立案機構（捐香油錢），**以便** 感恩神明的指引並支持系統的營運維護。
5. **會員註冊與登入**
   - **作為** 系統使用者，**我希望** 能建立專屬的個人帳號，**以便** 安全地管理我的歷史紀錄和個人化設定。
6. **每日運勢查詢 (加值功能)**
   - **作為** 經常登入的使用者，**我希望** 每天登入時能看到當日的專屬簡易運勢，**以便** 作為今日生活的參考。

## 3. 非功能需求

### 技術限制
- **前端框架**：HTML, CSS (Vanilla), JavaScript, Jinja2 模板系統
- **後端框架**：Python Flask
- **資料庫**：SQLite (適合小型伺服器與快速開發)

### 效能與安全考量
- **效能**：網頁載入速度需快速。抽籤過程若有動畫，應保持順暢，確保儀式感的沉浸體驗；API 讀寫時間應低於 1 秒。
- **安全**：
  - 密碼必須加密儲存（如使用 `werkzeug.security`）。
  - 若有金流（捐香油錢）應使用可信賴的第三方金流服務商，避免儲存非必要的信用卡資訊。
  - 需要防護常見的網路攻擊（如 SQL Injection、XSS），保護使用者儲存的算命結果隱私。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須有 - 核心流程)
- 會員註冊、登入與登出功能。
- 線上抽籤功能（包含題庫與結果配對）。
- 查詢詳細解籤內容。
- 檢視並儲存登入者的歷史算命紀錄。

### Should Have (應該有 - 完善體驗)
- 捐香油錢的頁面展示（初期可為單純的匯款資訊或純展示意向的按鈕）。
- 個人首頁顯示最新一次的抽籤結果。

### Nice to Have (最好有 - 加值功能)
- 豐富的抽籤儀式動畫（如搖籤筒特效）。
- 每日運勢推播或查詢。
- 將求籤結果分享至社群平台的功能。
